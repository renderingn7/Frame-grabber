# Frame Grabber

Извлекает кадр из каждого видео в папке на заданной секунде и сохраняет картинку рядом с видеофайлом.

Удобно, когда нужно быстро получить превью или один и тот же момент сразу из десятков роликов — не открывая каждый вручную.

## Что делает

- Берёт все видео из указанной папки и достаёт из каждого один кадр на нужной секунде.
- Сохраняет кадр в формате JPG рядом с видео, с тем же именем (`clip.mp4` → `clip.jpg`).
- Работает пакетно: одна команда — скриншоты сразу со всех файлов.

## Требования

- **Windows** (это `.bat`-файл).
- **ffmpeg** — бесплатная программа для работы с видео. Скрипт сам попытается её установить, если её ещё нет (см. ниже), так что обычно ничего настраивать не нужно.

## Как пользоваться

1. Скачайте файл `Frame-grabber.bat` и запустите его двойным кликом.
2. Когда откроется окно, **перетащите в него папку с видео** и нажмите Enter (или вставьте путь к папке вручную).
3. Введите **секунду**, на которой нужно сделать кадр — целое или дробное число, например `10` или `1.5`. Нажмите Enter.
4. Скрипт пройдёт по всем видео и сохранит кадры. Готовые `.jpg` появятся в той же папке, рядом с видеофайлами.

## Про ffmpeg (важно)

Для работы нужен ffmpeg. Скрипт ищет его в таком порядке:

1. Если ffmpeg уже установлен в системе — использует его.
2. Если рядом со скриптом лежит файл `ffmpeg.exe` — использует его.
3. Если не нашёл — **попытается автоматически установить ffmpeg** через `winget` (встроенный установщик Windows). Поэтому при первом запуске скрипт может что-то скачивать из интернета — это нормально.

Если автоустановка не сработала, сделайте одно из двух:

- Скачайте ffmpeg с [ffmpeg.org](https://ffmpeg.org/download.html) и положите файл `ffmpeg.exe` **рядом со скриптом**, либо
- Установите вручную командой: `winget install ffmpeg`

## Поддерживаемые форматы

`mp4`, `mkv`, `avi`, `mov`, `wmv`, `flv`, `webm`, `m4v`, `mpeg`, `mpg`, `ts`

## Замечания

- Если видео короче указанной секунды, для него кадр не создастся — в окне будет помечено как ошибка, остальные файлы обработаются как обычно.
- В конце работы скрипт покажет, сколько кадров сделано успешно и сколько с ошибками.
